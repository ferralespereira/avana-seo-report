used across the site are DEAD (present in the HTML, no CSS behind them).
This was the hidden root cause behind several of the visual bugs and it
changes how all future markup must be written. See "Key Discoveries".

    carried header="true", pushing it into the featured band, which

    as plain text. Deleted the unwrapped copy.

    Diagnosed with headless-Chrome measurement; the tables' min-content

    was a dead class. Fixed with table-layout:fixed and inline styles.

  - Hardcoded "localhost:7156" in live SEO markup. A JSON-LD FAQPage url

4. KEY DISCOVERIES (carry these forward)
------------------------------------------------------------------------

TAILWIND IS PRE-COMPILED OUTSIDE THIS PROJECT. The CSS is built by a
separate Astro project and copied into wwwroot/themes/maatdesk/. A
utility class only works if the Astro build already emitted it; any other
class is dead - it appears in the HTML and does nothing.

  Confirmed dead:  overflow-x-auto, border-collapse, list-decimal, pr-4,
                   my-4, my-8, space-y-1, bg-maat-green/5, border-l-4, and
                   the entire "maat-green" color (text/bg/border).
  Confirmed live:  maat-dark (brand green ~#3ED79F), maat-light (light
                   tint), maat-text (dark navy body/heading), maat-gray.

  Rule going forward: use bundle-backed classes or inline styles. Note
  the trap: "maat-green" reads like a valid brand class and is not.

THE FEATURED BAND HOLDS EXACTLY THREE POSTS. header="true" moves a post
into the band; the template renders nothing unless there are exactly
three. Fewer or more, and posts silently disappear from /blog. Any future
header="true" edit must keep the count at exactly three.

BLOG SORTING is by "datecreated", not "datepublished". Setting a sort
position therefore changes the date shown on the card.

SOURCE OF TRUTH for posts is App_Data\BlogPostEntities.xml.
