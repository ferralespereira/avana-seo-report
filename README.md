# Avana SEO Audit Report — Project README

## Project Overview

This project generates a comprehensive **SEO Audit Report** for two medical/aesthetic clinic websites managed by **Sevenlogik**:

1. **Avana Plastic Surgery** (`avanaplasticsurgery.com`) — Primary priority
2. **Avana Wellness Plus** (`avanawellnessplus.com`) — Secondary site

The report is prepared by **Javier Ferrales (SEO Specialist)** for presentation to **Dr. Labrador** and **Oreste Duarte (SEO Manager, Sevenlogik)**.

---

## How It Works

The entire report is generated by a single Node.js script:

```bash
node seo_report_v2.js
```

This produces `SEO_Audit_Report_Avana_v2.docx` — a fully formatted, professional Word document.

**Workflow:**
```
Edit seo_report_v2.js → node seo_report_v2.js → SEO_Audit_Report_Avana_v2.docx
```

## How to Update the Report

### Adding new data
1. Find the relevant section in `seo_report_v2.js` (search by section name e.g. "2.5 Backlink")
2. Replace `[ Fill — ... ]` placeholders with real data
3. Run `node seo_report_v2.js` to regenerate the `.docx`

### Adding a new section
1. Use the existing helper functions: `h1()`, `h2()`, `h3()`, `body()`, `bullet()`, `dataTable()`, `kpiTable()`, `kwTable()`, `findingsTable()`, `insight()`
2. Add a `pageBreak()` before major sections
3. Update the TOC page numbers after rebuilding

### Helper functions reference

| Function | Purpose |
|----------|---------|
| `h1(text)` | Main section heading (navy, large) |
| `h2(text)` | Subsection heading (teal) |
| `h3(text)` | Sub-subsection heading (navy, smaller) |
| `body(text)` | Regular paragraph text |
| `bullet(text)` | Bullet point |
| `italic(text)` | Italic note / data source label |
| `field(label, value, source)` | Single data field with label and source |
| `insight(text, type)` | Colored callout box — type: 'good', 'warn', 'bad', 'info' |
| `kpiTable(kpis[])` | Row of KPI cards — each has label, value, source |
| `dataTable(rows[][])` | General purpose table with header row |
| `kwTable(rows[][])` | Keyword table — 5 cols: keyword, position, volume, KD, source |
| `findingsTable(rows[][])` | Findings table — 4 cols: area, finding, priority, recommendation |
| `pageBreak()` | Insert a page break |
| `sectionDivider(label)` | Full-width navy banner for major section dividers |
| `spacer(n)` | Add n blank lines |

### Color coding for `insight()` boxes
- `'good'` → Green background — positive finding
- `'warn'` → Yellow background — warning / opportunity
- `'bad'` → Red background — critical issue
- `'info'` → Blue background — neutral information

### Priority levels for `findingsTable()`
- `"High"` → Red — fix immediately
- `"Medium"` → Yellow — fix within 30 days
- `"Low"` → Green — fix within 90 days
